【题解】LG1415 拆分数列

发表于 2018-10-10 | 修改时间: 2020-04-07 | 分类于 题解
Words count in article: 186 character(s) | Reading time ≈ 1 minute(s)

题目在这里.

可以看到，这题要求我们分割之后的各个数字满足以下几个条件。

  1. 先要满足最后一个数最小
  2. 如果有多组解，则使得第一个数尽量大；如果仍有多组解，则使得第二个数尽量大，依次类推……

因此，我们一个个地考虑，需要 DP 两次，每一次分别求出一个条件满足时的相关信息，再综合它们，得出答案。

第一次 DP 求出最后一个数最小可能是多少。

第二次 DP 求出字典序最大的答案。

设 $g(x)=\text{不考虑前 }i-1\text{ 个数，此时第一个数最大可能的值}$。记字符串长度为 $n$.

此时，由于我们无论如何也要保证最后一个数字为 $f(n)$, 所以这里 $g(x-\text{len}(f(n))) = f(n)$. 这里 $\text{len}(x)$ 代表 $x$ 在十进制表示下去掉前导 0 之后的位数。然后，从 $x-\text{len}(f(n))-1$ 开始往下动规。方程为

最后我们输出答案，假定我们现在已经处理到了原串的第 $i$ 位，那么只用看看 $g(i)$, 就知道下面应该输出什么数，输出完这个数之后，将当前指针后移，直到整个字符串扫完，这么做一定是正确的，因为 $g(i)$ 的定义就是如此，如果函数 $g$ 全部计算正确，就不会出错。特别地，个别数可能包含了前导 0 ，这个时候我们要先输出这些 0 ，再输出 $g(i)$. 判定的方法也很简单，如果当前指针指定的那个数不是 $g(i)$ 的第一位，那么它一定是一个 0 ，此时输出 0 ，后移一位指针，重复这个操作即可。

最后，由于中间计算的这个数可能非常大，所以还需要套上一个高精，总时间复杂度介于 $O(N^2)$ 和 $O(N^3)$ 之间，实际运行效率接近于 $O(N^2)$.
